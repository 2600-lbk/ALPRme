# Testing

## Test structure

```
test/
├── alpr/
│   ├── parity.test.ts              # Single-model regression against expected.json
│   ├── cross-model-parity.test.ts  # All 6 model combos vs Python results
│   ├── preprocess.test.ts          # Letterbox and coordinate mapping
│   └── decode.test.ts              # OCR argmax decode and IoU utility
├── capture/
│   ├── controller.test.ts          # CaptureController tile dispatch + reprojection
│   ├── tiling.test.ts              # generateTiles, reprojectBbox, crossTileNMS
│   ├── capabilities.test.ts        # buildInitialConstraints, getCapabilities
│   └── defaults.test.ts            # buildDefaultProfile, migrateRecord
├── e2e/
│   └── parity-report.test.ts       # Playwright: direct JS vs worker parity in browser
├── fixtures/
│   ├── pull_dataset.py             # Download USA plates from HuggingFace
│   ├── build_fixtures.py           # Generate expected.json with Python FastALPR
│   └── eval_parity.py              # Cross-model evaluation (6 combos × 4 strategies)
├── pipeline/                        # Prefilter, stabilizer, motion, dedup unit tests
├── storage/                         # IndexedDB schema, dedup logic
├── sensors/                         # Heading correction, declination
└── geomag/                          # Magnetic model evaluation
```

## Running tests

```bash
npm test                              # vitest run (all unit + parity)
npm run test:e2e                      # Playwright browser tests
npx vitest run test/alpr/parity.test.ts  # Single-model regression only
npx vitest run test/alpr/cross-model-parity.test.ts  # Multi-model JS parity
```

Python fixture tooling:

```bash
uv run python test/fixtures/pull_dataset.py     # Download USA test images
uv run python test/fixtures/build_fixtures.py   # Generate expected.json fixtures
uv run python test/fixtures/eval_parity.py      # Cross-model evaluation
```

## Single-model parity (`parity.test.ts`)

Runs the JS ALPR pipeline (direct, no worker) against `*.expected.json` fixtures
generated by the Python `fast-alpr` reference library. Uses the default model
combo (yolo-v9-t-384 + cct-xs-v2-global).

Assertions:
- Plate text edit distance ≤ 1 (bilinear resize differences between JS canvas
  and Python OpenCV can flip argmax on borderline character positions)
- Bounding box IoU ≥ 0.7 against reference
- Detector and OCR confidence in valid range [0, 1]
- Blank (all-black) image returns zero detections
- Corrupt model URL throws on init

## Cross-model parity (`cross-model-parity.test.ts`)

Runs all 6 model combinations (3 detectors × 2 OCRs) via the JS ALPR pipeline
and compares against Python results from `eval_results.json`. Requires
`eval_parity.py` to have been run first.

## Python cross-model evaluation (`eval_parity.py`)

Evaluates all 6 combos across 4 tiling strategies (whole-image, 1×1, 2×2, 3×3
grid) on every fixture image. Prints a console table with exact match rates,
edit distances, and timings. Writes `eval_results.json` for the JS test to
consume.

## Playwright E2E (`parity-report.test.ts`)

Loads a browser page with both the direct `Alpr` class and the `AlprClient`
(Web Worker). Runs both pipelines on each expected.json fixture and asserts
that:
- Direct and worker results match each other exactly (cross-match 100%)
- Each matches expected within 2 misses

## Test environment

Vitest runs in `jsdom` with a mocked canvas/OffscreenCanvas backed by the
`canvas` npm package (Cairo-based real rendering). This enables pixel-level
parity testing without a browser. The `test/vitest.setup.ts` file wires up
the mocks.

ONNX models live in `public/models/` and are loaded via absolute file paths
in vitest (e.g., `join(process.cwd(), 'public', 'models', 'yolo-v9-t-384.onnx')`).
